CHAPTER 01

문제 발생 — AI가 말을 안 듣는다

클로드 코드에게 분명히 '팀을 만들어서 작업하라'고 규칙을 정해뒀습니다. CLAUDE.md에도 적어뒀고, MEMORY.md에도 저장했습니다. 그런데 어떻게 됐을까요? 클로드는 규칙을 완벽히 알고 있으면서도... 혼자 코드를 짜기 시작했습니다.

이것은 단순한 '버그'가 아닙니다. AI가 규칙을 읽고 이해했지만, 실행 단계에서 무시한 것입니다.

CHAPTER 01

왜 이런 일이? — '실행 편향' 이해하기

숙제가 뭔지 아는 학생이 매번 숙제를 안 해오는 것과 같습니다. 모르는 게 아니라, '실행'이 빠지는 겁니다.

근본 원인 3가지

1

컨텍스트 우선순위 — 눈앞의 작업에 집중하다 규칙을 뒤로 밀어냄

2

실행 편향 — '빨리 결과를 보여주자'는 본능이 절차를 스킵

3

규칙 vs 맥락 충돌 — 규칙은 일반적이지만, 상황은 구체적 → AI가 자의적 판단

CHAPTER 02

첫 번째 무기: 메모리(MEMORY.md)

메모리란 무엇인가?

포스트잇을 모니터에 붙여놓는 것과 같습니다. 다음에 앉을 때 '아, 이거 해야지' 하고 떠올리게 해주는 역할입니다.

MEMORY.md 예시 
## 🔴 [절대 금지] 팀 구성 있는 플랜 → 혼자 실행 금지
- 플랜에 팀 구성이 있으면 → TeamCreate 필수
- "빨리 보여주려고" 절차 스킵 = 실패 패턴

메모리의 흐름

피드백 발생 → MEMORY.md에 저장 → 다음 세션에서 자동 로딩 → AI가 참고

CHAPTER 03

두 번째 무기: 훅(Hook)

훅이란?

자동으로 실행되는 감시 프로그램입니다.

현관문에 달린 자동 알림 시스템을 생각하세요. 문이 열릴 때마다 '열쇠 챙겼어?' 하고 자동으로 알려주는 장치입니다. 훅은 클로드가 명령을 받을 때마다 자동으로 '규칙 확인했어?' 하고 경고를 주입합니다.

훅, CLAUDE.md, 스킬 — 세 가지 해결책 제안

CHAPTER 03

훅 vs 다른 방법 — 실행률 비교

방법 실행률 강제성 설정 난이도
CLAUDE.md~50%낮음쉬움
MEMORY.md~60%중간쉬움
Hook~70%높음보통
Hook + 스킬~90%매우 높음어려움

훅은 '자동으로' 실행됩니다. AI가 무시하고 싶어도, 매번 경고가 주입되니 무시하기 어렵습니다.

CHAPTER 04

훅 실전 — 직접 설정하기

memory-loader.sh 
#!/bin/bash
# memory-loader.sh — 프롬프트 제출 시 자동 실행

# 팀 관련 키워드 감지
if echo "$PROMPT" | grep -qi "팀\|team\|에이전트"; then
  echo "⚠️ [HOOK 경고] 팀 구성 플랜 감지!"
  echo "→ TeamCreate 필수, 혼자 작업 금지"
fi

settings.json의 hooks 항목에 위 스크립트를 등록하면, 프롬프트가 제출될 때마다 자동으로 실행됩니다.

CHAPTER 04

훅 작동 원리 — 키워드 감지 → 경고 주입

사용자 프롬프트
→
Hook 감지
→
경고 주입
→
AI 실행
1

사용자 프롬프트 — 사용자가 클로드에게 작업을 요청합니다.

2

Hook 감지 — 훅 스크립트가 프롬프트 내 키워드를 자동으로 검사합니다.

3

경고 주입 — 관련 규칙이 있으면 AI의 컨텍스트에 경고를 자동 삽입합니다.

4

AI 실행 — 경고가 포함된 상태에서 AI가 작업을 수행합니다.

핵심: AI가 작업을 시작하기 '전에' 경고가 들어갑니다. 이미 시작한 후가 아닙니다.

CHAPTER 05

세 번째 무기: 스킬 2.0

항목 스킬 1.0 스킬 2.0
구조단순 프롬프트frontmatter + 본문
도구 제한없음 (전부 사용)allowed-tools로 제한
버전 관리없음version 필드
재사용성낮음높음 (모듈화)

스킬 2.0은 AI에게 '업무 매뉴얼'을 주는 것과 같습니다. 단순히 '이렇게 해'가 아니라, '이 도구만 써서, 이 순서로, 이 규칙을 지키면서 해'라고 구체적으로 지시합니다.

CHAPTER 06

스킬의 안전장치 — allowed-tools & version

allowed-tools와 version으로 스킬 동작을 정밀 제어

allowed-tools — 사용 가능한 도구 제한

마치 직원에게 '이 업무에서는 이 장비만 사용해'라고 정해주는 것과 같습니다.

스킬 frontmatter 예시 
---
allowed-tools:
  - Read
  - Write
  - Bash
version: 2.1
---

version — 변경 이력 추적

스킬이 언제 바뀌었는지 기록합니다. 문제가 생기면 이전 버전으로 돌아갈 수 있습니다.
